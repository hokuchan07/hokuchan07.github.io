PROBLEM 1
Cursorの使い方が分からない
📍 原因
フォルダ構造が個人ごとにバラバラだった。CLAUDE.md（監督の指示書）が無い場所で作業していたので、AIに何ができるか自体が見えていなかった
✅ 解決
全員が同じ .code-workspace でサイドバーを統一。フォルダ役割をサッカー例えで一覧化。「AIへの聞き方5パターン」を手元に。
→ 「どこで何をすればいいか」が見えるようになる
解決するPart: Part 3 / Part 6 / Part 7
PROBLEM 2
更新しても下に戻る
📍 原因
デスクトップなど git管理外のフォルダで作業していた。または git commit + push をしていなかったので、他の人の pull で上書きされた
✅ 解決
C:\git\ 配下のrepoルートでのみ作業。編集後すぐ git commit + push。さらに 夜20時に自動pushでセーフティネット。
→ 自分の変更が消えない仕組みが物理的にできる
解決するPart: Part 2 / Part 6
PROBLEM 3
AIに指示しても全く効かない
📍 原因
CLAUDE.md（監督の試合前指示書）が 無いフォルダでAIに頼んでいた。だからAIは「会社のルール」を一切読まずに動いていた
✅ 解決
.code-workspace で開くと必ずrepoルートに居る = CLAUDE.md が自動で効く。AIは毎回これを読んでから動く。
→ AIに依頼すれば想定通りの場所に・想定通りの形式で保存される
解決するPart: Part 3 / Part 6
PROBLEM 4
齋藤さんと競合する、自分の修正が消える
📍 原因
同じファイルを別々のタイミングで編集 → 後にpushした方が勝つ「後勝ち上書き」。下書きと完成版が同じ場所に置かれていたので、誰が触っていいのか分からなかった
✅ 解決
朝9時 自動pull / 夜20時 自動pushで時間帯を分離。個人作業は _local/ で隔離（他人に絶対影響しない）。下書きが整ってから昇格させる運用。
→ ぶつかる構造そのものが消える
解決するPart: Part 4 / Part 6
PROBLEM 5
どこに何があるか分からない、保存場所が不明
📍 原因
デスクトップ・ドキュメント・OneDriveに業務ファイルが散乱。フォルダの役割が共通理解されていなかった（「articles って何入れるの？」状態）
✅ 解決
C:\git\ に統一 + 1ウィンドウ統合で「ここ以外で作業しない」状態。フォルダ役割をサッカー例えで明確化（試合レポート＝articles/ など）。
→ 迷わずファイルを保存できる
解決するPart: Part 2 / Part 3 / 全体
PROBLEM 6
江口・齋藤さんが「外部」扱いになっている
📍 原因
GitHub上では Outside Collaborator（外部協力者）として個別repoに直付与されている状態。組織のメンバーリストに出ない、チーム権限管理ができない
✅ 解決
研修中に Organization Member に昇格招待。チームベースの権限管理（it-write / hr-input / contracts-read）に揃える。契約書の権限はwrite→readに是正
→ 社員として正式に扱われる、権限管理が一元化

共通エリア vs 個人エリア

⚽ 共通エリア

チーム全員が見る戦術ボード

  • 📂 各repoのルート〜サブフォルダ
  • ✏️ 編集すると 全社員のPCに反映
  • 📝 完成した記事・最終版資料を置く
  • 📊 月報・KPI・契約書（最終版）
  • 🔄 朝9時pull / 夜20時push で自動同期
📓 個人エリア

自分だけの自主練ノート

  • 📂 各repo内の _local/ 配下のみ
  • 🚫 GitHubに上がらない（自PCのみ）
  • 📝 下書き、頭の整理、実験
  • 💡 「これメモっとこ」「あとで整える」系
  • 👁️ 他人に見られない（共有も不可）

迷ったらこれ — 黄金ルール

こういう時は…どこに置く？
完成した記事・最終版の資料共通エリア（articles/ reports/ など）
下書き、まだ整っていない_local/ 配下
ふと思いついたメモ_local/ 配下
他人に見せたくない検討中の内容_local/ 配下
個人の好み・カスタマイズ設定_local/ 配下
みんなで使うテンプレ・チェックリスト共通エリア（templates/ など）

PART 2 10-25分

📁 フォルダ統一：C:\git\ へ揃える

なぜ C:\git\ という場所なのか？
OneDriveとgitは「ファイルをバックアップする似た仕組み」を持つ別の管理人。
同じファイルを2人の管理人が同時に触ると喧嘩して、ファイルが壊れます。
だから OneDriveが触らない場所（C:\git\） を新しく作って、gitだけに管理させます。

❌ NG：今までの場所

デスクトップ / ドキュメント / OneDrive配下

OneDriveが裏で自動同期している場所。ここにgitフォルダを置くと…

  • OneDriveがファイルを掴んでしまい git が動かない
  • ファイル名末尾に「 2.md」「 3.md」が勝手に増える
  • 同期タイミングで 「更新しても下に戻る」事件
  • 同じファイルが2箇所に存在して混乱
✅ OK：今日からの場所

C:\git\（Cドライブ直下のgit専用フォルダ）

OneDriveが触れない、誰の管理下でもない場所。ここに置くと…

  • gitだけが管理する 綺麗な状態
  • ファイルが壊れない、勝手に増えない、消えない
  • OneDriveとの喧嘩がそもそも発生しない
  • これまでの事故の9割が消える
💡 ひとことで言うと
OneDriveに「触らせない場所」を作って、そこでgitに専念させる、というだけの話。
場所を変えるだけで、これまで原因不明だった事故が一気に消えます。

PART 3 25-45分

🪟 1ウィンドウ統合：.code-workspace

なぜ「1ウィンドウ」にまとめるのか？
Cursorを開くと「机が1つ」画面に広がるイメージ。
AIは その机に置かれているノート（フォルダ）しか見えません。
別の机に置いてあるノートは「存在すら知らない」状態になります。

❌ NG：今までの開き方

4つのフォルダを別々のCursorウィンドウで開く

机が4つに分かれている状態。AIから見ると…

  • 「契約書どうなってた？」と聞いても、今の机のノートしか探さない
  • 別ウィンドウの契約フォルダの情報は 見えない
  • 「分かりません」と返ってきたり、間違った答えになる
  • 毎回ウィンドウを切り替える手間が発生
✅ OK：今日からの開き方

4つのフォルダを1つの机に並べる

.code-workspace ファイルをダブルクリックするだけ。すると…

  • 1ウィンドウの左サイドバーに 4フォルダが全部並ぶ
  • 「契約書どうなってた？」で 4つ全部を横断検索して答える
  • ウィンドウ切り替え不要、視線移動だけで全部見える
  • 毎回同じ机のセットがダブルクリックで開く
💡 .code-workspace ファイルとは？
「この机にこの4つのフォルダを並べてね」という設定書です。
ファイル名は runbird.code-workspace。これをダブルクリックするだけで、毎回同じ4 repoが綺麗に並んだ状態でCursorが起動します。
