ナレッジフォルダを開く手順

前田さん向け — RUNBIRD福祉事業ナレッジAI
🪟 Windows版 所要時間: 約3分

このマニュアルで何ができるか

4月10日の研修でセットアップ済みの「ランバード福祉ナレッジ」を、もう一度Cursorで開き直す方法です。

前提: 4/10の研修時にセットアップ済み（GitHub登録＋クローン＋動作確認pushまで完了済み）。PCにはすでにフォルダが入っている状態から始めます。

⚠️ 最初に読んでください — 「Clone repo」は押さない

Cursorの起動画面には「Clone repo」という目立つボタンがありますが、前田さんは押さないでください。

4/10の研修で既にクローン済みのため、もう一度押すと同じフォルダが2つできてしまい、どちらが最新か分からなくなります。

Cursor起動画面のボタン 前田さんの場合
Recent（最近開いたフォルダ） ✅ ここに runbird-welfare-knowledge があれば1クリック
Open folder（フォルダを開く） ✅ Recentに無ければこちら
Clone repo（GitHubから取得） ❌ 押さない（2重クローンの原因）
1
フォルダの場所を思い出す

研修でクローンしたフォルダは、以下のどれかに入っているはずです。思い当たる場所をエクスプローラーで開いてみてください。

💡 よくある保存場所（上から順に確認）

  • C:\git\runbird-welfare-knowledge（推奨していた場所）
  • C:\Users\（ユーザー名）\runbird-welfare-knowledge
  • デスクトップ（C:\Users\（ユーザー名）\Desktop\runbird-welfare-knowledge）
  • C:\Users\（ユーザー名）\Documents\runbird-welfare-knowledge

見つからない場合: Windowsキー + E でエクスプローラーを開き、右上の検索欄に runbird-welfare-knowledge と入力して検索してください。

✅ フォルダが見つかったら、そのパスをメモしておきます（エクスプローラーの上部アドレスバーに表示されています）。
2
Cursorでそのフォルダを開く

Cursorを起動し、左上のメニューから:

  1. File（ファイル）→ Open Folder（フォルダを開く）
  2. Step 1で見つけたフォルダを選択
  3. 「フォルダーの選択」をクリック
  4. 信頼確認ダイアログが出たら 「Yes, I trust the authors」 をクリック
✅ 左サイドバーに以下が見えたら成功です。
ナレッジ/、README.md、.cursorrules
💡 前回開いたフォルダなら: Cursor起動時の「Recent」（最近使用したフォルダ）一覧にも表示されているので、そこからワンクリックで開けます。
3
最新版に更新する（必須）

研修以降に鈴木さん・星崎さん・他メンバーが追加したナレッジを取り込みます。Cursorのターミナルで:

ターミナルを開く: Ctrl + `（バッククォート、半角/全角キーの隣）

git pull

「Already up to date.」 または 新しいファイルのリストが表示されればOK。

💡 エラーが出る場合: 研修以降にローカルで何か変更してしまった可能性があります。エラーメッセージを鈴木に送ってください。

🗓 これからの日常運用

朝イチ
git pull（他メンバーの追加分を取り込む）
日中
右パネルでAIに質問 / ナレッジ編集
退勤前
編集した日だけ：
git add -A && git commit -m "内容" && git push

🆘 どうしてもフォルダが見つからない場合

新しいPCに変えた、または何らかの理由でフォルダが消えてしまった場合は、もう一度クローンからやり直せます。Cursorのターミナルで:

cd C:\ mkdir git cd git git clone https://github.com/runbird-welfare/runbird-welfare-knowledge.git

注意: OneDrive配下・ドキュメントフォルダには置かないでください（クラウド同期と競合してファイルが壊れます）。必ず C:\git\ 配下に置きます。
