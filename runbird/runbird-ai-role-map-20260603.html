RUNBIRD ／ AI活用 役割配置 提案

CursorとClaude Code、誰がどう使うか
── 組織図に役割を配置する

結論：本体は「ツール」ではなく 会社が貯めるナレッジ（GitHubのMarkdown）。 Cursorはそれを引く窓、Claude Codeは貯めて・まとめる頭脳。 プランは役職の高さではなく「使い方の強度」で決める。

2026-06-03 ／ 株式会社斗（hishaku）作成
1

CursorとClaude Codeをどう使い分けるか

この2つは「対立」ではなく「層が違う」。同じナレッジの上に、用途の違う2つの窓を付ける。

📚 社内SaaSの本体 ＝ ナレッジ（GitHubのMarkdown）
商談記録・FAQ・組織・売上…を蓄積。これが会社の資産
↑ 引く↑ 貯める・まとめる
窓A ／ 知識を引く

Cursor

  • 「うちは〇〇についてどんな知見ある？」をGUIで質問
  • リポジトリ全体を検索（@Codebase）
  • ターミナル不要 → 非エンジニアでも使える
  • ＝ 知識量の差を埋める消費者向け
窓B ／ 貯める・まとめる

Claude Code

  • 録音・メール・CSVを自動で整形してMD化
  • 横断要約・ツール作成・git・自動化（スキル/hook）
  • 多案件を並列で回す＝運用者の頭脳・OS
  • ＝ 知識を作る・回す側向け

Cursor

コードを書く現場＋知識を引く窓

AIエディタ。@Codebaseで蓄積を検索。GUIで誰でも使える。

向く人：知識を引く大多数のスタッフ

Claude Code

仕事をやり切る自律エージェント

ファイル操作・コマンド・git・MCP連携・並列処理まで自律実行。

向く人：知識を貯める・回す運用者

Claudeアプリ（claude.ai）

出先の相談相手

PCを触らない壁打ち・PDF分析・スマホでの戦略相談。

向く人：経営・出先での思考整理
2

プランは「役職」でなく「使い方の強度」で決める

社長だから一番高いMax、は間違い。AIをどれだけ自律的・連続的に酷使するかで決める。

判断軸： 「相談・閲覧・壁打ち」中心なら Pro で十分。 「コードを書く・投入パイプラインを回す・一日中Opusを走らせる」運用者だけ Max（約$100/月）。 使わない機能に課金しない＝過剰投資を避ける。
軽い・相談型
知識を引く／戦略の壁打ち／資料要約／出先で相談
→ Cursor／Claude Pro
重い・運用型
自動で貯める／まとめる／ツール作成／多案件を並列で回す
→ Claude Code（Max）

具体例 ── こういう時はこっち

「誰が・どんな時に・どっちを使うか」をシーンで示す。

🔎
営業の永野さん「この業種の商談、過去の勝ちパターンある？」
Cursor で @Codebase に質問 → 蓄積された商談ナレッジから即回答。ベテランの暗黙知を新人が引ける。
⚙️
齋藤さん「Plaudの録音600件を分類してナレッジ化したい」
Claude Code が自動で文字起こし整形・分類・MD保存・git push まで実行。運用者の自律エージェント。
💭
嶋田社長「移動中に来期の事業方針を壁打ちしたい」
Claudeアプリ でスマホから戦略相談・PDF要約。社長クローンの対話もここ。
📖
嶋田社長「うちのCTIの導入実績、数字どうだっけ？」
Cursor Pro でデスクからナレッジを引く。社長自身が「会社の記憶」に直接アクセス。
📊
中村さん「全事業部の売上を横断集計してダッシュボード化」
Claude Code（Max） でツール自作・自動更新。経営企画の運用者として回す。
🤝
福祉の須賀さん「支援記録の書き方の社内ルール確認したい」
Cursor でナレッジ参照（※個人情報の実データはAIに入れない＝設計補助に限定）。
