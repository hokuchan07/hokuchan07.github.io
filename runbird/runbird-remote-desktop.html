🖥️ Chrome リモートデスクトップで先方PCを直接操作（図解）

鈴木さんのMacから、先方のWindows画面をまるごと操作します。画面の中のCursorもそのまま動かせます（＝LINEで送ってコピペ、が不要に）。

🪟
先方のWindows中のCursorを直接動かす

ポイント リモートデスクトップは「先方の画面・キーボード・マウスを遠隔に伸ばす」仕組み。 先方のCursorを開いて、ターミナルでコマンドを打つ・gitを直す——全部、先方の席に座っているのと同じようにできます。

② 先方PC側の初期設定（各PCで1回だけ・5分）

先方のWindows PCで、Chrome（ブラウザ）を開いて作業します。

  1. 1Chromeで remotedesktop.google.com/access を開く
  2. 2鈴木さんのGoogleアカウント（s.hokuoto）でログイン（先方のアカウントではなく、鈴木さんのを使う＝鈴木さんがどのPCにも繋げるように）
  3. 3「リモートアクセスの設定」→ ダウンロードしてインストール（画面の指示通り）
  4. 4このPCの名前を付ける（例：齋藤PC 江口PC）
  5. 5PIN（6桁以上の暗証番号）を設定（鈴木さんが覚えておく）
  6. 6「オンライン」になれば完了 ✓
⚠️ 常時接続（無人アクセス）になるので、先方に「サポートのため遠隔で入れるようにします」と一言伝えて合意を取っておく。PINは鈴木さんだけが知っている状態に。

③ 鈴木さんのMacから接続する

  1. 1Macで remotedesktop.google.com/access を開く（同じ s.hokuoto でログイン）
  2. 2設定したPC（齋藤PCなど）が一覧に出る → クリック
  3. 3PINを入力 → 先方の画面が表示され、操作できる状態に
  4. 4先方の Cursorを開いて、いつも通り作業（ターミナル・git・編集）

これで「LINEで送る→コピペ」が消えます。鈴木さんが直接やる／Claude（私）が画面を見ながら正確なコマンドを出して鈴木さんが先方Cursorで実行——推測ではなく実際の出力を見ながら進められます。

④ 2つのモードの違い（参考）

モード使い方向き
リモートアクセス
（無人・推奨）		PINを1回設定すれば、鈴木さんがいつでも繋げる◎ 定期メンテ・突発対応
リモートサポート
（その都度）		先方がワンタイムコードを発行→鈴木さんが入力（毎回必要）△ 1回きりの応急

→ 繰り返し直すなら「リモートアクセス（無人）」を各PCに設定しておくのがベスト。

⑤ 注意点

  • 先方が同時にPCを使っていると画面を取り合うので、作業は先方が使っていない時間に（事前に一言）。
  • PINは第三者に教えない。鈴木さんだけが管理。
  • 先方PCには顧客情報があるため、遠隔操作する合意を取ってから設定する。
