R18 EC × カラーミー × 自動ブログ投稿 構成イメージ

カラーミーEC案件向け 自動ブログ投稿機能の実現方式について（株式会社斗 / 鈴木 北斗）

① 現状の前提 カラーミー単体だと自動ブログ投稿が直接刺さらない

自動ブログ投稿エンジン
記事生成 → 投稿API叩く
✕
カラーミーEC（R18）
商品/受注APIはあるが
ブログ投稿系のAPIは非公開
→ WordPress用に組まれた自動投稿スクリプトは、そのままカラーミーに繋がらない

② 提案構成 WordPress を別建てして「埋め込み」で見せる

① 自動投稿エンジン

記事生成・スケジュール投稿・SEO制御

  • キーワード × テンプレで自動生成
  • WordPress REST API で投稿
  • 他クライアントへも横展開可能な共通基盤

② WordPress（別ドメイン or サブドメイン）

ブログ本体の置き場所

  • blog.shop-name.com 等
  • 記事DB・SEO評価はここに溜まる
  • カラーミー本体とは別管理で安全

③ カラーミーEC

商品販売・カート・決済はそのまま

  • iframe / JS で WP の記事一覧を埋め込み
  • EC機能はカラーミーのまま
  • 記事クリック → ブログ側へ遷移
自動投稿エンジン 記事生成 → 投稿予約
→
WordPress（別建て） 記事公開・SEO蓄積
→
カラーミーEC 記事一覧を埋め込み表示
ポイント：「カラーミーに自動投稿を“接続”」ではなく、「WordPress を裏で動かして“表に貼る”」発想に切り替える。
EC機能はカラーミーのまま安全に運用しつつ、ブログSEOは別ドメインで積み上げられる構成。

③ 今回作るもの ／ 既存資産との切り分け

パーツ 今回の扱い 状況
自動投稿エンジン本体（記事生成 + 投稿スケジューラ） 新規構築 本案件で初号機として構築。今後の横展開も見据えた共通基盤として設計
WordPress 構築 + ブログ用テーマ調整 新規構築 別ドメイン or サブドメインに設置、ブログ特化テーマで構築
WordPress 投稿スクリプト（REST API 連携部） 既存資産流用 当社で既に稼働している WordPress 自動投稿スクリプトを土台に拡張
カラーミー側への iframe / JS 埋め込み 新規構築 サイトデザインに馴染ませる UI実装
カラーミー本体（カート / 決済 / 商品登録） 対象外 従来どおり別ラインで対応

④ 案件としての位置づけ

クライアントメリット WordPress を意識せず、カラーミーEC上にブログが自然に並ぶ。SEOは別ドメインで蓄積、本体ECは触らず安全に運用できる。
運用メリット 記事生成〜公開を自動化することで、運用工数を最小化。継続的にブログ更新が走り、SEOからの集客導線を強化できる。
納期 2〜3ヶ月で実装可能。WP構築・エンジンMVP・カラーミー側の埋め込み統合のスケジュールを組める。
拡張性 自動投稿エンジンは共通基盤として設計するため、他のカラーミー／WordPress 案件にも適用可能。
