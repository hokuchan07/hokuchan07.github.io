なぜ作るのか

「あの面談で○○さん何て言ってた？」「あの契約解約はいつまでに通知すれば？」という質問が、嶋田社長や中村さんに集中している状況を解消します。

面談記録・契約書を専用リポジトリに集約し、Cursor AI が出典付きで即答できる状態を作ります。

A：「中村さん、これどうなってた？」 → 中村が確認・回答 → 中村に質問が集中
B：「Cursor、これどうなってる？」 → 出典付きで即答 → 中村は本来業務に集中

2 つのリポジトリの違いを一目で

人事面談 repo

用途面談での発言・状況の蓄積
投入前田・中村・河合・江口（4名）
形式md（PLAUD文字起こし）
参照嶋田社長のCursorのみ
機密度★★★ 最高（経営判断専用）

契約書 repo

用途契約条件の即時確認
投入中村（一喜）さん
形式PDF + 解説md（必ずセット）
参照Cursor持ち全員（6名）
機密度★★ 高（契約条件含む）

ファイル名のルール

以下の場所・命名で保存してください。

最重要：PDFと解説mdは必ずセットで

Cursor AI は バイナリPDFを直接読み解けません。PDF原本だけでは「契約どうなってる？」に答えられず、検索しても何も出ません。条件をスペック表化した .md を必ず同じ場所・同じ名前（拡張子だけ違い）で添えてください。

ファイル配置

ナレッジ/契約書/{種別}/{相手方}/{YYYYMMDD}_{契約名}.pdf ナレッジ/契約書/{種別}/{相手方}/{YYYYMMDD}_{契約名}.md

種別は5つから選択：

  • 顧客契約 / 業務委託 / 雇用労務 / NDA / その他
2

解説mdに書くこと

AIが即答できるよう、契約条件をスペック表形式で抽出。

  • 契約期間・自動更新の有無
  • 解約通知の期限
  • 契約金額・支払サイト
  • 秘密保持期間・競業避止
  • 損害賠償の上限

承認・接続の手順（共通）

既存の福祉ナレッジ・IT営業ナレッジと同じ流れです。福祉でやったメンバーは1〜2を飛ばせます。

1. GitHub招待メールを承認する

  1. GitHubにログインした状態で、自分宛の招待メールを開く
  2. 「View invitation」または「招待を確認」をクリック
  3. 「Accept invitation」を押して承認完了

2. Cursorで該当リポジトリを開く

  1. Cursor（または VSCode）を起動
  2. 「Clone Git Repository」または git clone でリポジトリを取得
    git clone https://github.com/runbird-inc/runbird-hr-knowledge.git git clone https://github.com/runbird-inc/runbird-contracts.git
  3. 初回オープン時に「このフォルダのタスクを自動的に許可しますか？」と聞かれたら「Allow / 許可」を選択

    → 以降フォルダを開くたびに自動で git pull が走り、最新ナレッジが揃います

3. 質問するとき（嶋田社長 / 契約書参照メンバー）

  1. Cursor上部のチャットを開く
  2. そのまま日本語で質問する
  3. AIが ナレッジ/ 配下を参照して、出典ファイル名付きで回答

よくある質問

Q. PLAUDの文字起こしをそのまま貼り付けるだけでいい？

A. はい、ただし冒頭のメタ情報（被面談者・面談実施者・日時）だけは追加してください。AIが「これは誰の何月の面談か」を識別する手がかりになります。

Q. 古い契約書はどう扱う？

A. 削除せずに残します。終了済の契約は _終了 サフィックスを付けるなど、いつ・どんな条件だったかが分かる状態で保管。改定があった場合は新ファイル（YYYYMMDD_変更後）として追加します。

Q. 契約書の解説mdって、毎回手書き？

A. 上のテンプレートをコピーしてスペック表を埋めるだけです。1契約あたり 5〜10分程度で済みます。慣れたら Cursor に「このPDFから条件抽出して解説mdを作って」と頼むと下書きを生成できます（ただし最後は人の確認が必須）。

Q. 人事面談の内容、後で本人に開示することになったら？

A. その想定で「事実ベース・引用ベース」で記録してください。印象論や個人攻撃は避けます。GitHubの履歴は残るので、編集・削除しても証拠としての痕跡は残ります。

Q. AIが「ナレッジに該当情報が見つかりませんでした」と言ったら？

A. これは 正しい挙動 です。ナレッジに無いことを推測で答えるのを禁止しているためです。実データを追加してから再質問してください（人事 → 該当面談を投入、契約書 → 中村さんに依頼）。

Q. 既存の福祉ナレッジ・IT営業ナレッジと混ざらない？

A. リポジトリが別なので物理的に分離されています。Cursorで質問する際は、その時開いているフォルダのナレッジだけを参照します。例えば人事面談リポジトリを開いているときに「契約条件は？」と聞いても、契約書リポジトリの中身は参照されません（必要に応じてフォルダを切り替えてください）。
