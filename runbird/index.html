ワークスペース セットアップ（1コマンド）

下記コマンド1回実行で、自分の権限内のナレッジが全部 Cursor の1ウィンドウで開ける状態になります。

共通セットアップ（ほぼ全員これ）

Mac の ターミナル（または VSCode/Cursor のターミナル）で以下を実行：

bash <(curl -sL https://hokuchan07.github.io/runbird/runbird-setup.sh)

スクリプトが自動的に：

  • ~/Documents/runbird-knowledge/ を作成
  • あなたがアクセスできるナレッジを自動でclone（権限ないものはスキップ）
  • Cursor用の統合ワークスペースファイル runbird.code-workspace を生成

完了後、Cursorで File > Open Workspace from File から runbird.code-workspace を開けば1ウィンドウで全部見えます。

事前条件：GitHub のOrg招待メール（runbird-inc）を承認しておくこと。Cursor で GitHub にサインインしておくこと（Settings → Accounts）。

よくある質問

Q. 既存のフォルダはどうなる？

A. 既にcloneしてあるフォルダはそのまま動きます（GitHubが旧URLから新URLへ自動リダイレクト）。整理したければ ~/Documents/runbird-knowledge/ にまとめ直すと統合ワークスペースで使えるようになります。

Q. Org招待メールはどこから来る？

A. 件名「[GitHub] @hokuchan07 invited you to join the runbird-inc organization」で届きます。緑のボタン押して承認するだけ。

Q. Cursorで質問するときの注意は？

A. 統合ワークスペースを開いた状態で質問すると、AIがどのナレッジを見るか自動的に判断します。「契約のことだけ知りたい」場合は、左サイドバーで「契約書」フォルダを選んでから質問するとピンポイントになります。

Q. 新メンバーを追加したい

A. 鈴木（斗）にChatworkで「○○さんに××のアクセスを」と依頼してください。Team割当てで完結します。
