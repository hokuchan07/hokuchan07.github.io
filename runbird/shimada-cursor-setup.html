「保存して」が魔法のキーワード

普通に質問するだけならその場で答えるだけですが、語尾に「保存して」「人事に保存」「契約に保存」を添えると、回答内容がmdファイルとして該当フォルダに自動で書き出されます。後から振り返り・検索できる蓄積資産になります。

フォルダ構造

嶋田社長のMacにはこの構造ができます。「黄色＝参照元（編集しない）」「緑＝出力先（蓄積される）」と覚えてください。

~/Documents/shimada-workspace/（Cursorで開くルート）

ナレッジ/ は触らない

「ナレッジ/」配下は前田・中村・河合・江口の4名が投入した内容を読み込む参照専用エリアです。Cursorで開くたびに自動で最新化（git pull）されます。社長は編集せず、参照だけしてください。
蓄積していくのは「人事/」「契約/」の2つだけ。これは社長専用です。

使用例

実際にどんな感じになるか、2つの典型例を見てください。

例1：人事面談の確認

中村は最近モチベーションについて何と言ってる？人事に保存しといて。
中村さんは2026-05-12の1on1で「全社AI化の進捗を見て自分のスキルがついていくか不安」と発言。具体的には新規開発に時間が取られず、新技術キャッチアップの時間が足りないとのこと。一方で、嶋田社長への信頼感と仕事への前向きさは維持。
参照: ナレッジ/runbird-hr-knowledge/ナレッジ/面談記録/2026-05/中村_1on1_20260512.md
保存しました： 人事/20260501_中村_モチベ確認.md

例2：契約条件の確認

X社との業務委託、解約はいつまでに通知すればいい？契約に保存。
X社業務委託契約は終了日の90日前までに書面通知が必要。自動更新条項あり（90日前までに通知が無ければ1年自動更新）。契約終了日は 2027-03-31 なので、解約する場合は 2026-12-31までに書面送付 が期限です。
参照: ナレッジ/runbird-contracts/ナレッジ/契約書/顧客契約/X社/20260315_業務委託契約書.md, .pdf
保存しました： 契約/20260501_X社契約_解約条件.md

「保存して」のバリエーション（どれでもOK）

人事に保存
→ 人事/ に書き出される
契約に保存
→ 契約/ に書き出される
メモして
→ AIが内容で判定して人事or契約へ
記録して
→ 同上
ファイルに残して
→ 同上
（何も付けない）
→ その場で答えるだけ・保存しない

セットアップ

この章は初回1回だけの作業です。鈴木（斗）が嶋田社長のMacで実施するので、社長は使い方の章までスキップで構いません。

初回セットアップ手順鈴木が実施

1

嶋田社長のMacのターミナルで以下を実行

1コマンドで全部が揃います（フォルダ作成・clone・ルール配置・自動pull設定）。

bash <(curl -sL https://hokuchan07.github.io/runbird/shimada-workspace-setup.sh)
2

Cursorで嶋田社長のGitHubアカウントにログイン

private repoにアクセスするため、Cursorの設定画面から GitHub 連携が必要。「Settings → Accounts → Sign in with GitHub」で shimada-runbird アカウントでログイン。

3

Cursorで「~/Documents/shimada-workspace」を開く

「File → Open Folder」で選択。初回オープン時に「タスクの自動実行を許可しますか？」が出たら必ず 「Allow」 を選ぶ（毎回の自動pullに必要）。

4

動作確認

Cursorのチャットを開いて、テスト質問してみる。例えば「今このワークスペースで参照できるナレッジを教えて」と聞き、runbird-hr-knowledge と runbird-contracts が両方答えられればOK。

+

セットアップが終わったら社長へ渡すもの

1. このガイドのURL（このページ）
2. デスクトップに shimada-workspace のショートカット（Cursorアイコンに紐付け推奨）
3. 「Cursorを開いて質問してください」の一言

使い方（嶋田社長の日常運用）

セットアップ完了後の日常的な操作はこれだけです。

毎日の使い方嶋田社長

1

Cursorを起動して shimada-workspace を開く

起動するだけで自動的にナレッジが最新化されます（裏でgit pullが走る）。

2

左サイドバーまたは下部の「Chat」を開く

キーボードショートカット：Cmd + L（Macの場合）

3

普通に日本語で質問する

例：

  • 「先月の面談で退職リスクが言及されたメンバーは？」
  • 「○○社との契約、解約はいつまでに通知すれば？」
  • 「今期評価で特に気になるメンバーはいる？」
4

記録を残したいときは語尾に 「保存して」 を添える

例：「中村のモチベ状況、人事に保存して」
→ 人事/20260501_中村_モチベ状況.md に自動で書き出される。

5

蓄積されたメモは 人事/ 契約/ から確認できる

左サイドバーのファイルツリーから直接開いて読める。後から「先月のメモ見たい」と言うだけでも検索可能。

★

やってはいけないこと

「ナレッジ/」配下のファイルを編集しない。これは前田・中村・河合・江口が投入したオリジナルです。誤って編集してもgit pullで戻りますが、競合エラーが出るので避けてください。
編集していいのは「人事/」「契約/」の2つだけです。

困ったとき

AIが「ナレッジに該当情報がありません」と返してくる
これは正しい挙動です。ナレッジに無いことを推測で答えるのを禁止しているため。前田・中村・河合・江口に「○○の面談記録を投入してほしい」と伝えるか、中村に「○○社の契約書を入れてほしい」と依頼してください。
「保存して」と言ったのにファイルが作られない
下のチャット欄でAIから「保存しますか？」のような確認が出ていることがあります。「Yes」または「Apply」を押すと作成されます。それでもダメな場合は .cursor/rules/extract-to-personal.mdc が読み込まれていない可能性があるため、鈴木にご連絡ください。
ナレッジが古い気がする
Cursorを一度閉じて開き直してください（自動pullが走ります）。または下部ターミナルで cd ナレッジ/runbird-hr-knowledge && git pull を実行。
人事の質問なのに「契約/」に保存されちゃった（or 逆）
「人事に保存」「契約に保存」と明示的に指定すれば確実に分類されます。曖昧な指示（ただ「保存して」）の場合、AIが内容で判定するので時々間違えます。間違えた場合はファイルを正しいフォルダへドラッグ＆ドロップで移動できます。
セットアップしてないMac/別の端末から見たい
外出先のPCやスマホからは Cursor PWA（cursor.com/agents） 経由で同じリポジトリに接続して質問できます。ただし「保存」機能はローカルのMacでのみ動作します。
機密情報がうっかり外部に漏れない？
このワークスペースは全部ローカルです。「人事/」「契約/」は完全にローカル保存（GitHub等にはアップしない設計）。「ナレッジ/」のリポジトリはGitHub privateですが、Cursor連携でAnthropicのAPI経由でやり取りする際は機密情報がそこに送られるため、Cursor設定の「Privacy Mode」を有効化することを推奨します（鈴木が初回設定時に有効化しています）。
