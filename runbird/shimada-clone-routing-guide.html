SHIMADA CLONE — ROUTING GUIDE

共有するもの・しないもの
どこに保存するか迷わない3つの軸

クローンに渡す情報を選別する判断基準。社員の自走を促しつつ、機密や個人情報を守る運用ルールです。

CORE PRINCIPLE

デフォルトは「社長専用」
「共有して」は意識して付け加えるキーワード

普通に質問して保存するだけなら自動的に 人事/契約/経営戦略/資産運用 のいずれかに保存され、誰にも見られません。
社員に渡したい時だけ 「共有して」 を語尾に添える、それだけです。

3 AXES

共有OKかどうかの3つの軸

質問に答える時、頭の中で以下の3つをチェックしてください。3つ全部YESなら共有OK、1つでもNGなら社長専用に保存。

1

社員に「役立つ」か

社員が判断に迷った時、これがあれば自走できる？

  • 同じことを何度も質問されるテーマ
  • 判断基準を社員が知ると意思決定が速くなる
  • 新人教育に使える
  • 業界知見・市場動向
2

漏れても「困らない」か

競合や外部に見られても問題ない？

  • 個人を特定する情報が含まれない
  • 取引先名で問題が出ない
  • 給与・評価・モチベが含まれない
  • 株主・M&A情報が含まれない
3

一般化「できる」か

個別の人や取引先の名前を抜いても意味が成立する？

  • 「判断の基準」だけ抽出できる
  • 「○○さんが」を「営業担当が」に置換可能
  • 「△△社が」を「取引先が」に置換可能
  • 抽象化しても価値が残る

判定式

3つ全部 YES → 共有OK（「共有して」で `共有/` に保存）
1つでも NO → 社長専用に保存（人事 / 契約 / 経営戦略 / 資産運用）

GRAY ZONE

迷った時の判断

原則：迷ったら社長専用に保存

後から「これは共有してよかった」と思ったら、社長専用フォルダから 共有/ にコピーすればOKです。逆は怖いです — 共有/ に上げて社員が見た後、削除しても「見られた事実」は消せません。

覚えておく：「共有→社長専用」は不可逆、「社長専用→共有」は後からいつでも可能。デフォルトを社長専用にしておくと安全。

グレーゾーンの典型パターン

個人特定 vs 一般化：「中村さんへの対応」は人事/、「中堅社員への対応の一般論」は共有/。同じ内容でも、個人名を抜いて一般化すれば共有可能になることが多いです。

取引先個別 vs 業界知見：「○○社との契約」は契約/、「同業界の標準的な契約条件」は共有/。

個別失敗 vs 組織の学び：「○○さんが原因の失敗」は人事/、「組織として学ぶべき構造」は共有/。

CURSOR USAGE

Cursorでの言い方一覧

語尾に付けるキーワードで保存先が振り分けられます。デフォルトは社長専用、共有したい時だけ明示。

キーワード 保存先 公開範囲
保存して 人事/ または 契約/（自動判定） 社長のみ
人事に保存 人事/ 社長のみ
契約に保存 契約/ 社長のみ
経営戦略に保存 経営戦略/ 社長のみ
資産運用に保存 資産運用/ 社長のみ
共有して 共有/{領域別サブフォルダ}/ 全社員参照可
MONTHLY REVIEW

月次レビューでの見直し

クローンを賢く育てるための月1ループ。鈴木と30分一緒にやります。

毎月のクローン育成サイクル

クローンは導入直後は精度が低めです。月次で社員の質問ログをレビューし、答えられなかった質問を社長が「共有して」で補強することで、月単位で賢くなります。

1
共有/ の追加内容チェック

1ヶ月で追加されたファイルの妥当性確認

2
答えられなかった質問特定

クローンが「該当情報なし」と返した質問を抽出

3
判断軸を補強

不足判断を社長が「共有して」で1分追記
