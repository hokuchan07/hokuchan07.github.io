福祉ナレッジAI セットアップガイド

使う人向け（豊田さん・遠藤さん他） — RUNBIRD福祉事業
使う人 所要時間: 約30分

このガイドの読み方

RUNBIRD福祉事業の142事業所のChatWork履歴・返信例・公式資料・加算情報をAI（Cursor）で検索・質問できる仕組みを、自分のPCに導入する手順です。

ゴール: Cursorを開いて、ナレッジAIに質問できる状態

📋 「使う人」の役割と権限

できること できないこと
✅ ナレッジを読む
✅ AIに業務質問する
✅ 加算・返信例・事業所履歴を検索する
✅ 自分のPC内で内容をコピー・参考にする 		❌ ファイルを直接編集して反映する
❌ 他の人のPCに変更を送る（push）

※ ナレッジに追加したい情報がある場合は、ChatWorkで管理者（鈴木・前田など）に依頼してください。後述の「編集したい時」セクション参照。

📦 セットアップ手順（初回のみ）

1
GitHub Org招待を受諾する

GitHubから招待メール（「runbird-welfare invited you to join...」）が届いています。

  1. 受信メールのリンク or 以下URLにアクセス:
    https://github.com/orgs/runbird-welfare/invitation
  2. 緑の 「Join runbird-welfare」 ボタンをクリック
✅ これでrunbird-welfareの「Member」になりました（読み取り権限付与）
2
Cursorをインストール

ブラウザで以下を開いてダウンロード:

https://www.cursor.com/download

OSに合わせて選択:

  • Windows: 「Download for Windows」
  • Mac (Apple Silicon=M1/M2/M3): 「Apple Silicon」
  • Mac (Intel): 「Intel」

ダウンロードしたインストーラーを実行 → 案内に従ってインストール → Cursorを起動

初回起動時、ログイン画面で 「Sign up」 または 「Sign in with Google」 でログイン。

💡 メールアドレスは個人/会社どちらでもOK。後でTeam招待を受ければランバードのTeamに紐づきます。
3
Gitをインストール

Windowsの場合:

https://git-scm.com/download/win

ダウンロードしたインストーラーを実行 → すべてデフォルトのまま「Next」連打でOK

Macの場合: macOS標準の「ターミナル」を開いて:

xcode-select --install

「インストール」をクリック → 5〜10分待つ

4
ナレッジリポジトリをクローン

Cursorを起動 → ターミナルを開く（Ctrl + `）

Windowsの場合:

cd C:\ mkdir git cd git git clone https://github.com/runbird-welfare/runbird-welfare-knowledge.git

Macの場合:

mkdir -p ~/git cd ~/git git clone https://github.com/runbird-welfare/runbird-welfare-knowledge.git
⚠️ 認証画面が出たら: ブラウザでGitHubにログインして「Authorize」をクリック。1回承認すれば次回から不要。
⚠️ 保存先の重要な注意:
Windowsの「ドキュメント」「OneDrive」フォルダ、Macの「書類」「デスクトップ」「iCloud Drive」配下には絶対に置かないでください。クラウド同期と競合してファイルが壊れます。必ず上記の C:\git または ~/git に置く。
✅ 「Receiving objects: 100% ...」と出れば成功です。
5
Cursorでフォルダを開く

Cursor画面の左上から:

  1. File → Open Folder
  2. クローンしたフォルダを選択:
    • Windows: C:\git\runbird-welfare-knowledge
    • Mac: ~/git/runbird-welfare-knowledge
  3. 「Yes, I trust the authors」をクリック

左サイドバーに以下の構成が見えれば成功です:

runbird-welfare-knowledge/ ├── .cursorrules ├── README.md └── ナレッジ/ ├── Chatwork履歴/ ├── 事業所/（142ファイル） ├── 加算/ ├── 公式資料/ ├── 返信例/ └── 法令/

💡 普段の使い方

🤖 AIに質問する（メインの使い方）

Cursor画面右側の 「New Agent」 入力欄に質問を入力 → Enter

質問例:

処遇改善加算が令和8年にどう変わるか教えて
利用者の家族から「対応が遅い」とクレームがあった時の返信例を作って
@ナレッジ/事業所/Royal.md この事業所の融資状況は？

※ @ を使うと特定のファイルを参照させられます

💡 朝イチに必ず実行: Cursorのターミナルで git pull → 管理者が前日までに追加した最新のナレッジが取り込まれます。

✏️ ナレッジを追加・編集したい時

2つのルート

使う人は直接の編集権限がないので、以下のどちらかで対応します。

A. 簡単ルート推奨

ChatWorkで管理者に依頼

「○○の情報を追加してほしい」とChatWorkで鈴木さん・前田さんなどに送る → 管理者が代理でナレッジに追加。

最も簡単。技術知識不要。

B. 上級ルート（任意）

Fork & Pull Request

GitHubで自分用のコピー（Fork）を作って編集 → PRで本家に提案 → 管理者が承認・マージ。

Git操作の基礎知識が必要。慣れてから挑戦。

💡 当面はAルート（ChatWorkで依頼）で十分です。業務が回りはじめてから、Bルートを学んでいけばOK。

🆘 困ったときは

よくあるトラブル

症状 対処
git コマンドが認識されない Step 3 のGitインストールがまだ → インストール後Cursorを再起動
クローン時に Permission denied エラー GitHub Org招待をまだ承諾していない可能性 → Step 1を再確認
クローン時に「Repository not found」エラー 同上（招待未承諾だと、privateリポジトリは見えない）
git pull で rejected エラー 理論上は起きない（Read権限ならpushしないため）→ エラー内容を鈴木さんに送る
AIが的外れな回答をする @ でファイル名を指定して質問し直す（例: @ナレッジ/加算/...）
その他のトラブル エラーメッセージのスクリーンショットを撮って ChatWork で鈴木に送ってください
